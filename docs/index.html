Objective

Design and characterise an affordable, extensible bioreactor monitoring platform for pharmaceutical-grade microbial culture research. The system logs temperature, relative humidity, atmospheric pressure, volatile organic compounds (VOC proxy via gas resistance), and spectral light intensity at 1 Hz, streaming data from an Arduino over serial to a host PC. The longer-term mission — informed by the Literature section below — is to demonstrate a monitoring architecture applicable to distributed and off-planet biomanufacturing: small, low-power reactors capable of producing biologics at a point-of-care or aboard a spacecraft, where traditional pharmaceutical supply chains are unavailable.

Three studies with E. coli have been run. Study 001 established baseline performance of the SHT30 + BME688 #1 stack over 12 sessions (~111 h logged). Study 002 added a second BME688 positioned at the bioreactor exhaust outlet and the AS7341 spectral sensor in its final sessions, running 9 sessions (~23 h logged). Study 003 continues with the full sensor stack (SHT30, dual BME688, AS7341), logging 6 sessions (~6 h logged) as of 2026-06-26. This sensor placement is critical to interpretation: SHT30 and BME688 #1 measure bench-ambient conditions, while BME688 #2 samples exhaust gas — thermally and chemically closer to the culture headspace.

The work is directed toward a peer-reviewed journal article on small-scale pharmaceutical bioreactors. Platform code, raw data, and this analysis dashboard are version-controlled in this repository.

Hardware

Arduino microcontroller (no on-board RTC) streams comma-separated sensor rows at 115,200 baud. All sensors share a single I²C bus. Absolute wall-clock timestamps are assigned by the host PC logger; the Arduino provides only millis()-based uptime. Two BME688 sensors occupy the same I²C bus via their address-select pin (SDO): BME688 #1 has SDO pulled to GND → address 0x76; BME688 #2 has SDO pulled to 3.3 V → address 0x77.

Study 002 sensor placement — key interpretation context

Sensor position is fixed across study 002 and directly affects how every chart should be read. This placement must be documented here because it is not encoded in the CSV data:

■ SHT30 (0x44)
On bench adjacent to bioreactor. Measures ambient room temperature and relative humidity. Capacitive polymer RH element; accuracy ±0.2 °C, ±2 % RH (typ.). Not in the culture headspace.
■ BME688 #1 (0x76)
On bench adjacent to bioreactor. Measures ambient T, RH, P, and background lab VOCs. Provides environmental baseline and drift reference for BME688 #2. MOX sensor requires 1–3 h burn-in per session.
■ BME688 #2 (0x77)
Positioned at the bioreactor exhaust outlet. Temperature ≈ headspace temperature. Humidity ≈ exhaust saturation (expect near 100 % RH). Gas resistance tracks culture-derived VOCs (acetate, ethanol, acetoin from overflow metabolism). This is the metabolically informative sensor.
■ AS7341 (0x39)
DFRobot Gravity module. Eight narrowband channels F1–F8 (415–680 nm) + CLEAR + NIR. Added in the final two sessions of study 002 — very few data points exist. Intended as an OD₆₀₀ proxy and pigment indicator; requires opaque vessel enclosure to eliminate ambient light contamination.

Software

Arduino Firmware

user_provided/arduino/01_stepper_motor/01_stepper_motor.ino — C/C++. Reads all sensors once per second, streams 22 comma-separated fields over serial at 115,200 baud, and manages peristaltic pump duty-cycle (10 min ON / 1 min OFF by default) and stepper motor speed. No RTC — uptime is millis() since boot.

Required Arduino libraries:

Wire (built-in) Adafruit_SHT31 Adafruit_BME680 DFRobot_AS7341

Python Data Loggers

Active scripts live in user_provided/arduino/01_stepper_motor/; archived copies in user_provided/python/archive/. Each script opens the Arduino serial port, rejects lines without exactly 22 fields (handles startup noise and truncated packets), prepends pc_timestamp and elapsed_s, and appends rows to a timestamped CSV in the correct studies/ subfolder.

Output path is resolved automatically — three levels up from the script file gives the repo root, then studies/STUDY_NAME/. Set STUDY_NAME and SERIAL_PORT at the top of the active script before each session. On Linux the port is typically /dev/ttyUSB0; on Windows COM3/COM4. Git remote is over SSH (git@github.com:jesnyder/astroPharmReactor.git) — HTTPS credential-manager is not installed on the Fedora host.

Data Pipeline

user_provided/python/generate_study_summaries.py — full pipeline script. Run via bash user_provided/makefile/build.sh which also opens the site. Key internal design decisions for future reference:

Schema normalisation: Five historical CSV versions (A–E) exist across the two studies as sensors were added incrementally. A COL_ALIASES dict in the script maps every legacy column name to a canonical name (e.g., temp_C → sht30_temp_C). All versions are processed identically after normalisation.

Bad-data filtering applies three classifications — nan, below_min, above_max — using per-column limits in SENSOR_LIMITS: temperature 0–80 °C, humidity 0–100 %, pressure 800–1,100 hPa, gas resistance 100–500,000,000 Ω. Filtered values are replaced with None (→ JSON null → Plotly line break).

Downsampling: step = max(1, N // MAX_CHART_POINTS) where MAX_CHART_POINTS = 4000. Gaps between sessions longer than 60 s receive a null-row separator so Plotly does not draw a connecting line across the gap.

description.txt: Place a plain-text file at studies/study00X_ecoli/description.txt — it is read by read_description() and shown verbatim at the top of the study block on the site.

Auto-interpretation: generate_interpretation() classifies temperature stability (range <2 °C / <5 °C / >5 °C), inter-sensor agreement (BME688 #1 vs #2 ΔT, SHT30 vs BME688 offset), gas resistance level and fractional variation, and bad-data fraction, producing a temp_pressure_text string and a list of bullets rendered below each study.

Each generated docs/js/study00X_ecoli.js populates two globals:

Sensor Physics & Bioreactor Thermodynamics

The sensors used in this system measure variables that are physically coupled through well-established relationships. Understanding these relationships is essential for separating true biological signals from sensor artefacts, environmental drift, and placement effects.

1  Ideal Gas Law — Temperature–Pressure Coupling

For an ideal gas in a closed, constant-volume vessel:

PV = nRT          [SI units: Pa, m³, mol, J·mol⁻¹·K⁻¹]

Constant n, V:    P₁/T₁ = P₂/T₂   →   dP/dT = P/T ≈ 3.4 hPa/K at STP

This reactor operates as an open vessel (gas exchange with atmosphere). For an open system at constant Patm, temperature and pressure are decoupled: culture heating does not drive pressure changes. Any apparent T–P correlation in the data reflects either meteorological pressure variation or BME688 temperature-compensation artefact — not culture physiology.

2  Saturation Vapour Pressure — Antoine Equation

The saturation vapour pressure of water P* as a function of temperature T:

log₁₀(P*) = A − B / (C + T)     [T in °C, P* in mmHg → ×1.33322 for hPa]

Antoine constants for water (1–100 °C; NIST Webbook):
  A = 8.07131,  B = 1730.63,  C = 233.426

Key values:
  P*(22 °C) ≈  26.4 hPa   (typical lab ambient)
  P*(37 °C) ≈  62.8 hPa   (E. coli culture temperature)
  Ratio: 2.38 — exhaust air can carry 2.4× more water vapour

Clausius-Clapeyron slope (liquid–vapour equilibrium):
  dP*/dT = Lᵥ · P* / (Rᵥ · T²)
  Lᵥ = 2.501 × 10⁶ J/kg  (latent heat of vaporisation)
  Rᵥ = 461.5 J/(kg·K)    (specific gas constant for water)
  At T = 300 K: dP*/dT ≈ 1.8 hPa/°C

This means the BME688 #2 sensor positioned at the bioreactor outlet (37 °C exhaust) should read substantially higher relative humidity than bench sensors at ambient temperature — even before biological off-gassing. Humidity readings approaching 100 % at the outlet indicate condensation, not accurate RH measurement.

3  BME688 Metal-Oxide Gas Sensor — Response Model

The BME688 uses a heated metal-oxide (MOX) sensing layer whose resistance changes in the presence of reducing gases:

R = R₀ · [CVOC]⁻ᵅ · exp(Eₐ / kT)

Parameters:
  R₀   — baseline resistance in clean air [Ω]
  α    ≈ 0.3–0.7   (power-law exponent; gas-dependent)
  Eₐ   ≈ 0.1–0.5 eV (activation energy for surface reaction)
  k    = 8.617 × 10⁻⁵ eV/K (Boltzmann constant)
  T    — sensor hot-plate temperature (~340 °C operational)

Sensitivity: dR/d[C] = −α · R₀ · [C]⁻⁽ᵅ⁺¹⁾   (compresses at high [C])

Key E. coli metabolites detected (all reduce R):
  • Acetic acid (CH₃COOH, pKₐ = 4.76) — overflow metabolism product
  • Ethanol (C₂H₅OH) — mixed acid fermentation by-product
  • Acetoin (C₄H₈O₂) — quorum-sensing signalling molecule

CO₂ is NOT detected by n-type MOX sensors (fully oxidised; no electron donation).

The sensor is non-selective: R integrates the response to all reducing gases simultaneously. Discrimination between metabolites requires temperature cycling or multi-sensor arrays (e-nose approach). Burn-in time of 1–3 h per session is required before R₀ stabilises.

4  E. coli Growth — Monod Kinetics

Biomass accumulation in a batch culture under substrate limitation follows Monod kinetics:

dX/dt = (μ(S) − D) · X

Monod equation:
  μ(S) = μ_max · S / (Kₛ + S)

Parameters for E. coli (aerobic, 37 °C, glucose):
  μ_max  ≈ 0.7–1.0 h⁻¹
  Kₛ     ≈ 0.02 g·L⁻¹  (glucose half-saturation constant)
  D      = dilution rate (= 0 for batch culture)

Overflow metabolism threshold:
  At μ > μ_crit ≈ 0.35 h⁻¹, acetate is excreted regardless of O₂ availability.
  Acetate excretion rate: qₐ ∝ (μ − μ_crit) · X

Implication: exponential-phase cultures operating above μ_crit will produce
  increasing acetate → decreasing BME688 gas resistance at the outlet.

5  Metabolic Heat & VOC Production

Aerobic respiration (complete):
  C₆H₁₂O₆ + 6 O₂  →  6 CO₂ + 6 H₂O      ΔG° = −2870 kJ/mol

Overflow metabolism (acetate excretion):
  C₆H₁₂O₆  →  2 CH₃COOH + 2 CO₂          (anaerobic-like even under aerobic conditions)

Calorimetric heat correlation (Minkevich & Eroshin, 1973):
  Q_bio ≈ 518 kJ per mol O₂ consumed

Respiratory quotient:
  RQ = mol CO₂ produced / mol O₂ consumed
  RQ = 1.0 (pure aerobic on glucose);  RQ > 1 indicates overflow metabolism

Practical link to sensor data:
  Rising cell density (OD₆₀₀ proxy: AS7341 counts ↓)
  ↓
  Rising metabolic activity → acetate accumulation in headspace
  ↓
  BME688 #2 resistance decreases (reducing VOC signal)
  ↓
  Outlet humidity rises (more evaporation, CO₂ stripping of dissolved water)

6  Henry's Law — Dissolved CO₂ and pH

Henry's Law (gas dissolution):
  p_CO₂ = Kₕ · [CO₂]_aq

  Kₕ(25 °C) ≈ 29.4 L·atm/mol  (CO₂, direct form)

CO₂ dissolution drives culture acidification:
  CO₂ + H₂O  ⇌  H₂CO₃  ⇌  H⁺ + HCO₃⁻     pKₐ₁ = 6.35

Agitation effect: mechanical mixing strips dissolved CO₂, shifting equilibrium
toward gas phase and partially buffering pH drop. In this open reactor, the
peristaltic pump provides limited gas exchange — adequate for small-volume cultures
but insufficient to maintain DO for high-density fermentation.
