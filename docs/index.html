Objective

Design and characterise an affordable, extensible environmental monitoring platform for observing fungal growth through a solid substrate. The system logs temperature, relative humidity, atmospheric pressure, volatile organic compounds (VOC proxy via gas resistance), and spectral light intensity every 3 seconds, streaming data from an Arduino over serial to a host PC. The goal is to find sensor-based proxies for substrate colonization Рђћ using ambient-vs-in-bag differentials in temperature, humidity, and gas resistance as indicators of fungal metabolic activity, and light transmission through the bag as an indicator of mycelial density.

Study 001 (pilot) began 2026-06-30. A liquid culture of fungus was inoculated into a bag of sterile substrate and outfitted with sensors the same day at 5:00pm. Sensor placement is critical to interpretation: SHT30 and BME688 #1 sit outside the bag, on the table next to it, and measure ambient room conditions. BME688 #2 sits inside the bag, in direct contact with the substrate headspace. The AS7341 light sensor sits under the bag, illuminated from above by a light strong enough to saturate the sensor (1000 across all channels) if the bag were not attenuating it Рђћ so light transmitted through the bag, not absolute brightness, is the signal of interest.

Platform code, raw data, and this analysis dashboard are version-controlled in this repository.

Hardware

Arduino microcontroller (no on-board RTC) streams comma-separated sensor rows at 115,200 baud. All sensors share a single I┬▓C bus. Absolute wall-clock timestamps are assigned by the host PC logger; the Arduino provides only millis()-based uptime. Two BME688 sensors occupy the same I┬▓C bus via their address-select pin (SDO): BME688 #1 has SDO pulled to GND Рєњ address 0x76; BME688 #2 has SDO pulled to 3.3 V Рєњ address 0x77.

Study 001 sensor placement Рђћ key interpretation context

Sensor position is fixed for study 001 and directly affects how every chart should be read. This placement must be documented here because it is not encoded in the CSV data:

Рќа SHT30 (0x44)
On the table, outside the bag. Measures ambient room temperature and relative humidity. Capacitive polymer RH element; accuracy ┬▒0.2 ┬░C, ┬▒2 % RH (typ.). Not in contact with the substrate.
Рќа BME688 #1 (0x76)
On the table, outside the bag, next to SHT30. Measures ambient T, RH, P, and background lab VOCs. Provides an environmental baseline and drift reference for BME688 #2. MOX sensor requires 1РђЊ3 h burn-in per session.
Рќа BME688 #2 (0x77)
Positioned inside the substrate bag, in the headspace above the substrate. Temperature and humidity reflect in-bag microclimate, not room conditions. Gas resistance tracks VOCs building up in the enclosed headspace, potentially including fungal metabolic off-gassing. This is the sensor most directly exposed to the fungal culture.
Рќа AS7341 (0x39)
DFRobot Gravity module. Eight narrowband channels F1РђЊF8 (415РђЊ680 nm) + CLEAR + NIR. Positioned under the bag, beneath a light source strong enough to saturate the sensor if unobstructed. Intended as a mycelial-colonization proxy: as the fungus colonizes the substrate and the bag interior fills with white mycelium, transmitted light reaching the sensor is expected to fall.

Wiring

All four sensors share a single I┬▓C bus (Arduino SDA/SCL Рђћ pins A4/A5 on an Uno-class board). SHT30 and the AS7341 have fixed I┬▓C addresses; the two BME688 sensors are otherwise identical modules distinguished only by the state of their SDO (address-select) pin Рђћ SDO tied to GND gives address 0x76 (BME688 #1), SDO tied to VCC gives 0x77 (BME688 #2). The pump and stepper are driven directly by digital/PWM output pins and are not on the I┬▓C bus.

Source: user_provided/arduino/01_stepper_motor/addresses.txt and the pin definitions in 01_stepper_motor.ino.

Software

Arduino Firmware

user_provided/arduino/01_stepper_motor/01_stepper_motor.ino Рђћ C/C++. Reads all sensors every 3 seconds, streams 22 comma-separated fields over serial at 115,200 baud, and manages peristaltic pump duty-cycle (0 min ON / 1000 min OFF by default Рђћ effectively disabled for Study 001) and stepper motor speed. No RTC Рђћ uptime is millis() since boot.

Required Arduino libraries:

Wire (built-in) Adafruit_SHT31 Adafruit_BME680 DFRobot_AS7341

Python Data Loggers

Active scripts live in user_provided/arduino/01_stepper_motor/; archived copies in user_provided/python/archive/. Each script opens the Arduino serial port, rejects lines without exactly 22 fields (handles startup noise and truncated packets), prepends pc_timestamp and elapsed_s, and appends rows to a timestamped CSV in the correct studies/ subfolder.

Output path is resolved automatically Рђћ three levels up from the script file gives the repo root, then studies/STUDY_NAME/. Set STUDY_NAME and SERIAL_PORT at the top of the active script before each session. On Linux the port is typically /dev/ttyUSB0; on Windows COM3/COM4.

Data Pipeline

user_provided/python/generate_study_summaries.py Рђћ full pipeline script. Run via bash user_provided/makefile/build.sh which also opens the site. Key internal design decisions for future reference:

Schema normalisation: Five historical CSV versions (AРђЊE) exist from the prior project on this rig, as sensors were added incrementally. A COL_ALIASES dict in the script maps every legacy column name to a canonical name (e.g., temp_C Рєњ sht30_temp_C). All versions are processed identically after normalisation.

Bad-data filtering applies three classifications Рђћ nan, below_min, above_max Рђћ using per-column limits in SENSOR_LIMITS: temperature 0РђЊ80 ┬░C, humidity 0РђЊ100 %, pressure 800РђЊ1,100 hPa, gas resistance 100РђЊ500,000,000 ╬Е. Filtered values are replaced with None (Рєњ JSON null Рєњ Plotly line break).

Downsampling: step = max(1, N // MAX_CHART_POINTS) where MAX_CHART_POINTS = 4000. Gaps between sessions longer than 60 s receive a null-row separator so Plotly does not draw a connecting line across the gap.

description.txt: Place a plain-text file at studies/study00X_pilot/description.txt Рђћ it is read by read_description() and shown verbatim at the top of the study block on the site.

Auto-interpretation: generate_interpretation() classifies temperature stability (range <2 ┬░C / <5 ┬░C / >5 ┬░C), inter-sensor agreement (BME688 #1 vs #2 ╬ћT, SHT30 vs BME688 offset), gas resistance level and fractional variation, and bad-data fraction, producing a temp_pressure_text string and a list of bullets rendered below each study.

Sensor Physics & Substrate Bag Thermodynamics

The sensors used in this system measure variables that are physically coupled through well-established relationships. Understanding these relationships is essential for separating true biological signals from sensor artefacts, environmental drift, and placement effects.

1  Ideal Gas Law Рђћ TemperatureРђЊPressure Coupling

For an ideal gas in a closed, constant-volume vessel:

PV = nRT          [SI units: Pa, m┬│, mol, J┬иmolРЂ╗┬╣┬иKРЂ╗┬╣]

Constant n, V:    PРѓЂ/TРѓЂ = PРѓѓ/TРѓѓ   Рєњ   dP/dT = P/T РЅѕ 3.4 hPa/K at STP

The substrate bag is not a sealed rigid vessel, so temperature and pressure inside it are not tightly coupled the way the ideal-gas relation above would predict for a closed system: any flexing or venting of the bag lets internal pressure track the surrounding atmosphere. Any apparent TРђЊP correlation between BME688 #1 (outside) and BME688 #2 (inside) most likely reflects meteorological pressure variation or a BME688 temperature-compensation artefact, not fungal activity Рђћ pressure is a poor proxy for what is happening inside the bag.

2  Saturation Vapour Pressure Рђћ Antoine Equation

The saturation vapour pressure of water P* as a function of temperature T:

logРѓЂРѓђ(P*) = A Рѕњ B / (C + T)     [T in ┬░C, P* in mmHg Рєњ ├Ќ1.33322 for hPa]

Antoine constants for water (1РђЊ100 ┬░C; NIST Webbook):
  A = 8.07131,  B = 1730.63,  C = 233.426

Key values:
  P*(22 ┬░C) РЅѕ 26.4 hPa   (typical ambient room temperature)
  P*(30 ┬░C) РЅѕ 42.4 hPa   (representative in-bag temperature)
  Ratio: 1.6 Рђћ a warmer bag interior can hold ~60% more water vapour before saturating

Clausius-Clapeyron slope (liquidРђЊvapour equilibrium):
  dP*/dT = LрхЦ ┬и P* / (RрхЦ ┬и T┬▓)
  LрхЦ = 2.501 ├Ќ 10РЂХ J/kg  (latent heat of vaporisation)
  RрхЦ = 461.5 J/(kg┬иK)    (specific gas constant for water)
  At T = 300 K: dP*/dT РЅѕ 1.8 hPa/┬░C

This means BME688 #2, sealed inside the substrate bag with a moist substrate, is expected to read substantially higher relative humidity than the bench sensors outside Рђћ this is normal for an enclosed, moist microclimate and does not by itself indicate fungal activity. Humidity readings at or near 100 % RH may reflect condensation on the sensor element rather than an accurate reading.

3  BME688 Metal-Oxide Gas Sensor Рђћ Response Model

The BME688 uses a heated metal-oxide (MOX) sensing layer whose resistance changes in the presence of reducing gases:

R = RРѓђ ┬и [CVOC]РЂ╗рхЁ ┬и exp(EРѓљ / kT)

Parameters:
  RРѓђ   Рђћ baseline resistance in clean air [╬Е]
  ╬▒    РЅѕ 0.3РђЊ0.7   (power-law exponent; gas-dependent)
  EРѓљ   РЅѕ 0.1РђЊ0.5 eV (activation energy for surface reaction)
  k    = 8.617 ├Ќ 10РЂ╗РЂх eV/K (Boltzmann constant)
  T    Рђћ sensor hot-plate temperature (~340 ┬░C operational)

Sensitivity: dR/d[C] = Рѕњ╬▒ ┬и RРѓђ ┬и [C]РЂ╗РЂйрхЁРЂ║┬╣РЂЙ   (compresses at high [C])

Fungal metabolism is a known source of reducing VOCs (e.g. ethanol and other
alcohols, aldehydes) Рђћ the specific compound mix depends heavily on species and
growth stage and is not characterised for this study. The sensor's response is a
non-selective aggregate signal, not a speciated measurement.

COРѓѓ is NOT detected by n-type MOX sensors (fully oxidised; no electron donation),
so respiration-driven COРѓѓ buildup in the bag headspace is invisible to this sensor.

The sensor is non-selective: R integrates the response to all reducing gases simultaneously and cannot on its own distinguish a fungal VOC from ambient lab solvents or cleaning agents. Discrimination between sources would require temperature cycling or multi-sensor arrays (e-nose approach). Burn-in time of 1РђЊ3 h per session is required before RРѓђ stabilises Рђћ early-session gas readings should be treated cautiously.
